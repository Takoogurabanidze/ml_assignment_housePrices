## კონკურსის მიმოხილვა

ამ Kaggle კონკურსის მიზანია სხვადასხვა მახასიათებლის მიხედვით საცხოვრებელი სახლების საბოლოო ფასის პროგნოზირება.

მონაცემები მოიცავს:
- რიცხვით ცვლადებს (მაგ: ფართობი, ოთახების რაოდენობა)
- კატეგორიულ ცვლადებს (მაგ: უბანი, სახლის ტიპი, მასალა)
- ხარისხობრივ შეფასებებს (მაგ: მდგომარეობა)

ამოცანა წარმოადგენს Regression ტიპის პრობლემას, სადაც სამიზნე ცვლადია SalePrice.

ამ პროექტში ჩემი მთავარი მიზანი იყო ისეთი მოდელის შექმნა, რომელიც კარგად განაზოგადებს მონაცემებს და არ გადავა overfitting-ში.

მიდგომა მოიცავდა შემდეგ ნაბიჯებს:

- Feature Selection — მნიშვნელოვანი feature-ების შერჩევა (Correlation, Lasso, RFE)
- სხვადასხვა მოდელების ტესტირება და მათი შედარება
- overfitting და underfitting-ის ანალიზი

საბოლოოდ საუკეთესო შედეგი მივიღე XGBoost მოდელით.

ყველა ექსპერიმენტი დავათრექე MLflow-ში, რაც დამეხმარა მოდელების სწორად შეფასებაში და შედარებაში.

## Feature Engineering

### კატეგორიული ცვლადების რიცხვითში გადაყვანა
კატეგორიული ცვლადების რიცხვითში გადაყვანა პროექტში განხორციელებულია რამდენიმე განსხვავებული მიდგომით, რაც დამოკიდებულია თითოეული ცვლადის ბუნებაზე და მის კავშირზე მიზნობრივ ცვლადთან.

იმ ცვლადებისთვის, სადაც არსებობს ბუნებრივი წესრიგი, გამოყენებულია Ordinal Encoding:

    qual_map = {
        "None": 0,
        "Po": 1,
        "Fa": 2,
        "TA": 3,
        "Gd": 4,
        "Ex": 5
    }

აქ მნიშვნელობებს შორის დაცულია თანმიმდევრობა, რაც მნიშვნელოვანია მოდელისთვის.

ზოგიერთ კატეგორიაზე გამოყენებულია ისეთი გადაყვანა, რომელიც ასახავს მათ გავლენას ფასზე. მაგალითად, ავტოფარეხის მდგომარეობა:

    garage_map = {
        "None": 0,
        "Unf": 1,
        "RFn": 2,
        "Fin": 3
    }

აქ "Unf" (unfinished) ითვლება უარესად, ვიდრე "Fin" (finished), რაც რეალურადაც აისახება ქონების ღირებულებაზე.

ორობითი (binary) გადაყვანა გამოყენებულია იმ ცვლადებისთვის, რომლებსაც მხოლოდ ორი მნიშვნელობა აქვთ, როგორიცაა:
CentralAir, Street

ასეთ შემთხვევაში მნიშვნელობები გარდაიქმნება 0 და 1-ად.

One-hot encoding გამოყენებულია იმ კატეგორიული ცვლადებისთვის, სადაც არ არსებობს მკაფიო წესრიგი ან დამოკიდებულება.

ამ მეთოდით თითოეული კატეგორია გარდაიქმნება ცალკე სვეტად, რაც მოდელს აძლევს საშუალებას დამოუკიდებლად შეაფასოს თითოეული მნიშვნელობა.

### NaN მნიშვნელობების დამუშავება
NaN მნიშვნელობების დამუშავება პროექტში განხორციელდა ცვლადის ტიპისა და მისი მნიშვნელობის გათვალისწინებით, რათა შენარჩუნებულიყო მონაცემების ლოგიკა და მაქსიმალურად გაზრდილიყო მოდელის სიზუსტე.

რიცხვითი (numeric) ცვლადებისთვის გამოყენებულია მედიანით შევსება. ეს მიდგომა არჩეულია იმიტომ, რომ მონაცემებში გვხვდება აუტლაიერები და მედიანა, საშუალოსთან შედარებით, ნაკლებად მგრძნობიარეა მათ მიმართ. შედეგად, შევსებული მნიშვნელობები უკეთ ასახავს მონაცემების რეალურ განაწილებას.

კატეგორიული ცვლადებისთვის გამოყენებულია ორი განსხვავებული სტრატეგია, მათი მნიშვნელობიდან გამომდინარე:

- მოდით (ყველაზე ხშირი მნიშვნელობით) შევსება გამოყენებულია იმ ცვლადებზე, სადაც გამოტოვებული მნიშვნელობა არ ნიშნავს კონკრეტული მახასიათებლის არარსებობას და უფრო ლოგიკურია მისი ჩანაცვლება ყველაზე გავრცელებული მნიშვნელობით. ეს მიდგომა საშუალებას აძლევს მოდელს არ დაკარგოს ინფორმაცია და არ დაამატოს ხელოვნური კატეგორიები.

- "None" მნიშვნელობით შევსება გამოყენებულია იმ ცვლადებზე, სადაც NaN რეალურად ნიშნავს შესაბამისი მახასიათებლის არარსებობას. მაგალითად, ავტოფარეხთან (Garage), ბუხართან (Fireplace), სარდაფთან (Basement) დაკავშირებულ ცვლადებში გამოტოვებული მნიშვნელობა ნიშნავს, რომ ეს ელემენტი საერთოდ არ არსებობს. ასეთ შემთხვევაში "None"-ის გამოყენება უფრო სწორად ასახავს რეალურ მდგომარეობას, ვიდრე მოდით შევსება.

ეს მიდგომების კომბინაცია უზრუნველყოფს, რომ მონაცემებში არ დარჩეს გამოტოვებული მნიშვნელობები, ხოლო თითოეული ცვლადი დამუშავდეს მისი შინაარსის შესაბამისად, რაც საბოლოოდ აუმჯობესებს მოდელის მუშაობას.

## Feature Selection

ამ ეტაპზე გამოვიყენე რამდენიმე განსხვავებული მიდგომა, რათა შემერჩია ყველაზე ინფორმაციული ცვლადები და შემემცირებინა ზედმეტი ან redundant features.

Correlation Filter
პირველ რიგში გამოვიყენე კორელაციის ანალიზი, რომლის მიზანი იყო ერთმანეთთან ძლიერად დაკავშირებული (highly correlated) ცვლადების აღმოჩენა.

ამოვიღე ერთმანეთთან ძალიან მაღალი კორელაციის მქონე feature-ები
მიზანი იყო რომ შემემცირებინა ისეთი კატეგორიები, რომლებიც იგივნეირ გავლენას ახდენდნენ მიზნობრივ ცვლადზე.

შემდეგ ეტაპზე გამოვიყენე mutual information და tree-based importance:

შევაფასე თითოეული feature-ის კავშირი target ცვლადთან
შევარჩიე ყველაზე ინფორმაციული ცვლადები

ამან საშუალება მომცა დამეტოვებინა მხოლოდ ის feature-ები, რომლებიც რეალურად ახდენენ გავლენას ფასზე.

Tree-based Feature Selection (Random Forest)
გამოვიყენე Random Forest მოდელი feature importance-ის დასათვლელად:

მოდელმა დაალაგა ცვლადები მნიშვნელობის მიხედვით
დავტოვე მხოლოდ ყველაზე ძლიერი feature-ები

საბოლოოდ კი დავტოვე 40 ყველაზე მნიშვნელოვანი მონაცემი:

მიზანი: feature-ების რაოდენობის შემცირება
ნაკლებად მნიშვნელოვანი ცვლადების მოცილება
მონაცემის უფრო კომპაქტურ და სტაბილურ ფორმაში გადაყვანა
