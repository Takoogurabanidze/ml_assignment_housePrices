House Prices - Predicting House Prices

---

## კონკურსის მიმოხილვა

ამ Kaggle კონკურსის მიზანია სხვადასხვა მახასიათებლის მიხედვით საცხოვრებელი სახლების საბოლოო ფასის პროგნოზირება.

მონაცემები მოიცავს:
- რიცხვით ცვლადებს (მაგ: ფართობი, ოთახების რაოდენობა)
- კატეგორიულ ცვლადებს (მაგ: უბანი, სახლის ტიპი, მასალა)
- ხარისხობრივ შეფასებებს (მაგ: მდგომარეობა)

ამოცანა წარმოადგენს Regression ტიპის პრობლემას, სადაც სამიზნე ცვლადია SalePrice.

---

## ჩემი მიდგომა

ამ პროექტში ჩემი მთავარი მიზანი იყო ისეთი მოდელის შექმნა, რომელიც კარგად განაზოგადებს მონაცემებს და არ გადავა overfitting-ში.

მიდგომა მოიცავდა შემდეგ ნაბიჯებს:

- მონაცემების გაწმენდა (NaN მნიშვნელობების დამუშავება და უსარგებლო სვეტების მოცილება)
- Feature Engineering — ახალი, უფრო ინფორმატიული ცვლადების შექმნა
- Feature Selection — მნიშვნელოვანი feature-ების შერჩევა (Correlation, Lasso, RFE)
- სხვადასხვა მოდელების ტესტირება და მათი შედარება
- overfitting და underfitting-ის ანალიზი

საბოლოოდ საუკეთესო შედეგი მივიღე XGBoost მოდელით.

ყველა ექსპერიმენტი დავათრექე MLflow-ში, რაც დამეხმარა მოდელების სწორად შეფასებაში და შედარებაში.

## Feature Engineering

### კატეგორიული ცვლადების რიცხვითში გადაყვანა
კატეგორიული ცვლადები გარდავქმენი რიცხვით ფორმატში სხვადასხვა მეთოდით (ordinal encoding, one-hot encoding), რათა მოდელს შეეძლოს მათი გამოყენება.

### NaN მნიშვნელობების დამუშავება
გამოტოვებული მნიშვნელობები შევავსე სხვადასხვა სტრატეგიით:
- რიცხვითი ცვლადები — მედიანით
- კატეგორიული ცვლადები — მოდით ან "None" მნიშვნელობით

### Cleaning მიდგომები
- მაღალი missing value-ის მქონე სვეტების მოცილება
- უსარგებლო და redundant feature-ების დადროპვა
- დაბალი ვარიაციის მქონე ცვლადების ამოღება

---

## Feature Selection

გამოვიყენე რამდენიმე მიდგომა:

- Correlation Filter — სუსტი კავშირის მქონე ცვლადების მოცილება
- Lasso Regression — ნაკლებად მნიშვნელოვანი feature-ების წონების ნულამდე დაყვანა
- RFE (Recursive Feature Elimination) — საუკეთესო feature-ების შერჩევა

შედეგად შევამცირე feature-ების რაოდენობა და გავაუმჯობესე მოდელის გენერალიზაცია.

---

## Training

### ტესტირებული მოდელები
- Linear Regression
- Decision Tree
- Random Forest
- XGBoost

### Hyperparameter ოპტიმიზაცია
მოდელებისთვის გამოვიყენე სხვადასხვა ჰიპერპარამეტრების კომბინაციები (depth, number of trees, learning rate და სხვა) საუკეთესო შედეგის მისაღებად.

### საბოლოო მოდელის შერჩევის დასაბუთება
საბოლოოდ ავარჩიე XGBoost, რადგან:
- ჰქონდა საუკეთესო სიზუსტე
- უკეთესად აბალანსებდა bias-ს და variance-ს
- ნაკლებად იყო მიდრეკილი overfitting-ისკენ
